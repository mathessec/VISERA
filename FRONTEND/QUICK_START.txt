STEP-BY-STEP INSTRUCTIONS TO RUN THE APPLICATION
================================================================================

STEP 1: START THE BACKEND
--------------------------
1. Open a terminal/command prompt
2. Navigate to backend directory:
   cd Full-Backend/backend

3. Run the backend:
   ./mvnw spring-boot:run
   
   OR on Windows:
   mvnw.cmd spring-boot:run

4. Wait for "Backend Application Started" message
5. Backend will run on: http://localhost:8081

STEP 2: START THE FRONTEND
---------------------------
1. Open a NEW terminal/command prompt
2. Navigate to frontend directory:
   cd FRONTEND
  // pnpm i
3. Start the development server:
   pnpm dev

4. Open your browser to:
   http://localhost:5173

5. You should see the Login page

STEP 3: CREATE TEST USERS
--------------------------
Use Postman, curl, or any REST client:

Create Admin User:
-------------------
POST http://localhost:8081/api/auth/register
Content-Type: application/json

{
  "name": "Admin User",
  "email": "admin@visera.com",
  "password": "admin123",
  "role": "ADMIN"
}

Create Supervisor User:
------------------------
POST http://localhost:8081/api/auth/register
Content-Type: application/json

{
  "name": "John Supervisor",
  "email": "supervisor@visera.com",
  "password": "super123",
  "role": "SUPERVISOR"
}

Create Worker User:
-------------------
POST http://localhost:8081/api/auth/register
Content-Type: application/json

{
  "name": "Mike Worker",
  "email": "worker@visera.com",
  "password": "worker123",
  "role": "WORKER"
}

STEP 4: LOGIN & EXPLORE
------------------------
1. Go to http://localhost:5173
2. Login with one of the created users
3. Based on role, you'll be redirected to:
   - admin@visera.com ‚Üí Admin Dashboard (with charts)
   - supervisor@visera.com ‚Üí Supervisor Dashboard (operations)
   - worker@visera.com ‚Üí Worker Dashboard (tasks)

STEP 5: TEST NAVIGATION
------------------------
- Click on sidebar menu items
- Test Products page
- Test SKUs page
- Test Shipments page
- Try logging out and logging in with different roles

APPLICATION FLOW
================================================================================

1. User Access Flow:
   User visits site ‚Üí Login page ‚Üí Enter credentials ‚Üí API call
   ‚Üí Backend validates ‚Üí Returns JWT token + role ‚Üí Store in localStorage
   ‚Üí Redirect to role-specific dashboard

2. Navigation Flow:
   Click menu item ‚Üí React Router navigates ‚Üí ProtectedRoute checks auth
   ‚Üí Validate role ‚Üí Render Layout + Page ‚Üí Display content

3. Data Fetching Flow:
   Page mounts ‚Üí useEffect triggers ‚Üí Call service function ‚Üí API request
   ‚Üí Request interceptor adds JWT token ‚Üí Backend processes ‚Üí Response
   ‚Üí Response interceptor handles errors ‚Üí Update state ‚Üí UI re-renders

4. Logout Flow:
   Click logout ‚Üí Remove tokens from localStorage ‚Üí Redirect to login

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "Network Error" when logging in
Solution: Ensure backend is running on port 8081

Problem: "401 Unauthorized"
Solution: Token expired, logout and login again

Problem: Page shows nothing after login
Solution: Check browser console for errors, ensure role is set correctly

Problem: Sidebar not showing correct menu items
Solution: Verify user role in localStorage (F12 ‚Üí Application ‚Üí Local Storage)

Problem: Products/SKUs/Shipments page empty
Solution: Create test data using API endpoints

Problem: Build fails
Solution: Run "pnpm install" again and "pnpm run build"
